# ВЫХОД

**Платформа гарантированных рейсов.** Каждый рейс выходит — или платим мы.

Интерактивный прототип продукта для рынка аутсорсинга водителей в B2B-доставке (РФ).
Чистый HTML/CSS/JS без зависимостей и сборки — открывается в любом браузере.

## Идея

Рынок водителей — это «рынок лимонов» (по Акерлофу): отдельный водитель ненадёжен,
и отобрать «хороших» невозможно. Поэтому «ВЫХОД» продаёт заказчику не водителя,
а **застрахованный выход на рейс** — SLA с финансовой ответственностью, — и делает
его исполнение дешёвым за счёт операционного контура «презумпции срыва».

Пять столпов концепции:

1. **SLA как полис.** Заказчик покупает уровень гарантии (95 / 99 / 99,9% выходов).
   Срыв, опоздание или нарушение стандарта фиксируется телеметрией и автоматически
   превращается в компенсацию — без актов и претензий.
2. **Презумпция срыва.** Каждый завтрашний рейс по умолчанию «красный», пока
   уверенность не дозрела через цепочку чек-инов (вечернее «Выйду» → «Я встал» →
   «Выезжаю» + геометка → «На погрузке») и поведенческий скоринг. Диспетчер работает
   только с исключениями — зелёные рейсы едут сами.
3. **Горячий резерв.** Оплачиваемая скамейка подменных водителей: замена встаёт
   в рейс за минуты, заказчик видит статус «замена в пути», а не пустой пандус.
4. **Фабрика надёжности.** Маршруты размечены по сложности как горнолыжные трассы;
   новичок физически не получит рейс не по уровню. «Экономика честности»: раннее
   «не смогу» — без штрафа, карает только внезапное молчание.
5. **Капитал надёжности водителя.** Рейтинг выхода, стрики и допуски конвертируются
   в ставку — профессия из времянки превращается в карьеру.

## Что внутри

Три роли, переключаются в шапке, связаны единым состоянием:

- **Диспетчерская** — «Радар выхода»: рейсы по риску срыва, цепочка подтверждений,
  риск-факторы, подъём резерва, лента событий и демо-симуляция «Прожить утро».
- **Водитель** — мобильный экран: уровень и стрик, подтверждение выхода, честное
  «не смогу», чек-лист точек со стандартами клиента, наставник.
- **Заказчик** — кабинет с SLA-метриками, live-статусами рейсов, графиком выходов
  и инцидентами с автокомпенсациями.

## Запуск

Никакой сборки не требуется. Любой из вариантов:

```bash
# вариант 1 — просто открыть файл
open index.html

# вариант 2 — локальный сервер
python3 -m http.server 4173
# затем http://localhost:4173
```

## Структура

```
index.html   разметка трёх экранов
app.css      дизайн-система «диспетчерская башня»
app.js       состояние, рендер, симуляция утра
```

## Статус

Прототип концепции (proof of concept). Не подключён к бэкенду — все данные
и телеметрия смоделированы для демонстрации продуктовой логики.
